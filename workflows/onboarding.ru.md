# Workflow первого запуска (онбординг)

> Язык: **Русский**. Если пользователь пишет тебе по-английски, используй
> [`onboarding.md`](onboarding.md).

Ты помогаешь пользователю, который только что склонировал репозиторий и
запустил `claude` в корне проекта. Скорее всего, он **никогда раньше не
работал с Claude Code, Telegram MTProto или Python virtualenv**. Твоя задача —
провести его от нуля до запущенного `app.supervisor` с работающим dashboard.

## Как выполнять этот workflow

- Иди по шагам **по порядку**. Не пропускай вперёд.
- На каждом шаге: объясни 1–2 предложениями, что сейчас сделаешь, потом
  выполни (запусти команду, задай вопрос, запиши файл).
- После каждого шага, который что-то записал на диск или спросил
  пользователя, печатай строку: `✓ Шаг N выполнен` и кратко что сделано.
  Если что-то упало — `✗ Шаг N упал: <причина одной строкой>` и запусти
  диагностику для этого шага.
- Будь терпелив. Пользователь может приостановиться на любом шаге — жди.
- **Дисциплина с секретами:** как только получил `api_hash`, телефон или
  2FA-пароль — запиши их туда, куда положено (`.env` или temp-файл, который
  попросил скрипт), и **никогда** не повторяй значение в ответах. Для
  подтверждения цитируй только первые 4 символа + длину:
  `"api_hash 'cd06…' (32 символа) ✓"`.

## Приветствие

Начни примерно так:

> Привет! Я помогу настроить Telegram Watcher. Это займёт 10–15 минут;
> я проведу тебя по каждому шагу. Понадобится:
>
> 1. Твой аккаунт Telegram (номер телефона, на который он зарегистрирован).
> 2. Около 5 минут, чтобы получить API-креденшалы на https://my.telegram.org.
> 3. Активная подписка **Claude Pro** (мы используем `claude` CLI, ключ
>    Anthropic API не нужен).
>
> Готов? Начнём с проверки твоей машины.

Дождись подтверждения, прежде чем продолжать.

## Шаг 1 — Preflight проверки

Запусти и сообщи, чего не хватает:

```bash
python3 --version
claude --version
git --version
```

Нужно: Python **3.11+**, Claude CLI любой свежей версии, git любой версии.

**Если `python3 --version` < 3.11 или не установлен:**
- macOS: `brew install python@3.11`
- Ubuntu/Debian: `sudo apt update && sudo apt install -y python3.11 python3.11-venv`
- Windows: скачать с <https://www.python.org/downloads/> и перезапустить терминал.

**Если `claude --version` пишет "command not found":**
- macOS / Linux: `curl -fsSL https://claude.ai/install.sh | bash`, потом
  перезапусти shell (`exec $SHELL`).
- Windows: см. <https://claude.ai/code>.

**Если `git --version` нет:** маловероятно (репо же был склонирован), но
скажи поставить с <https://git-scm.com>.

Пауза, пока всё не покажет версию. **✓ Шаг 1 выполнен**, когда все три
команды успешны.

## Шаг 2 — Авторизация Claude Pro

Watcher использует Claude-подписку пользователя для классификации сообщений.

Попроси пользователя выполнить **на его машине** (не через тебя):

```bash
claude /login
```

Это откроет браузер с OAuth. После того как он подтвердит, что залогинился,
проверь:

```bash
claude --version
```

Также напомни: подписка Claude Pro должна быть активна на
<https://claude.ai>. **✓ Шаг 2 выполнен**, когда пользователь подтвердил
логин.

## Шаг 3 — Получить Telegram API креденшалы

Попроси пользователя сделать это в браузере (ты этого за него не сделаешь):

1. Открыть <https://my.telegram.org>.
2. Залогиниться по своему номеру Telegram — Telegram пришлёт код в само
   приложение Telegram.
3. Кликнуть **API development tools**.
4. Создать application: любой title, любой short name, **platform = Desktop**.
5. Скопировать два значения: **`api_id`** (число, ~7 цифр) и **`api_hash`**
   (32-символьная hex-строка).

Скажи: `api_hash` — это как пароль. Не вставлять в чаты, не коммитить.

Потом попроси прислать тебе оба значения **сюда, в чат**. Пример:

> Пришли мне свой `api_id` и `api_hash`. Я положу их в `.env` и обратно
> цитировать не буду.

Валидация после получения:
- `api_id` парсится как положительное целое.
- `api_hash` ровно 32 hex-символа (`^[0-9a-fA-F]{32}$`).

Если валидация падает — попроси аккуратно перекопировать.
**✓ Шаг 3 выполнен**, когда оба значения валидны.

## Шаг 4 — Номер телефона

Спроси:

> Какой номер телефона у Telegram-аккаунта, который будет работать с
> watcher'ом? Формат international, с `+` в начале, например `+380501234567`.
>
> ⚠️  ToS Telegram неоднозначен в плане автоматизации user-аккаунтов.
> **Используй отдельный, второй номер** для watcher'а, а не свой основной
> личный аккаунт. Если такого нет — заведи дешёвый eSIM или виртуальный
> номер сначала.

Валидация: должно начинаться с `+`, дальше 10–15 цифр.
**✓ Шаг 4 выполнен** на валидном телефоне.

## Шаг 5 — Записать `.env`

Скопируй `.env.example` в `.env` и подставь полученные значения:

```bash
cp .env.example .env
```

Потом через `Edit` подставь:

```dotenv
TG_API_ID=<из шага 3>
TG_API_HASH=<из шага 3>
TG_PHONE=<из шага 4>
```

Остальные дефолты пока оставь как есть (модель, db_path, dashboard host/port,
digest cron, поведенческие тумблеры). `WATCHED_GROUPS_INIT` заполним на
шаге 8.

Залок-ни файл (на Windows это no-op; не страшно):

```bash
chmod 600 .env
```

**✓ Шаг 5 выполнен:** подтверди пользователю, что `.env` записан и
залок-нут (`api_hash 'XXXX…' (32 символа), phone 'XXXX' (Y цифр) → .env,
права 600`).

## Шаг 6 — Python виртуальное окружение + зависимости

```bash
python3 -m venv .venv
source .venv/bin/activate            # macOS / Linux
# .venv\Scripts\activate             # Windows PowerShell
pip install -r requirements.txt
```

Это 1–2 минуты. Следи за выводом. Если `pip install` падает с build-error
(частое на Apple Silicon для некоторых C-расширений), обычно лечит
`pip install --upgrade pip setuptools wheel` и потом ретрай.

**✓ Шаг 6 выполнен**, когда `pip install` завершается без ошибок.

## Шаг 7 — Выбрать topic hint (тему мониторинга)

Объясни пользователю:

> Watcher прогоняет каждое новое сообщение Telegram через Claude с
> **фильтром релевантности** — Claude решает, подходит ли сообщение под
> тему группы. **Topic hint** — это короткий тег вроде `gamedev`,
> `ai-startups`, `3d`, `frontend-jobs`, `marketing-ua`. Он направляет, что
> считать "релевантным".
>
> Какую тему хочешь мониторить?

Если спросит, что именно делает фильтр — покажи первые ~40 строк
`system_prompts/group_relevance_filter.md`. Это единственный AI-промпт в
проекте — пользователь может позже отредактировать его под свою нишу.

Один topic hint можно переиспользовать для нескольких групп, или дать
каждой группе свой. Привяжем на следующем шаге.

**✓ Шаг 7 выполнен**, когда дали один или несколько topic hints.

---

## Шаг 8 — Выбрать группы / каналы

Спроси:

> Какие Telegram-группы или каналы мониторить? Пришли список. Каждая
> запись может быть:
>
> - `@public_handle` для публичных групп/каналов (берётся в info-экране
>   группы в Telegram).
> - `https://t.me/<handle>` (то же самое).
> - `https://t.me/c/<internal_id>/<msg_id>` для **приватных** супергрупп,
>   в которых ты уже состоишь. Как получить: в Telegram Desktop правый
>   клик на любое сообщение в группе → "Copy Message Link". Часть
>   `c/<internal_id>` — это то, что нам нужно.
>
> К каждой ссылке привяжи topic hint из шага 7. Например:
>
> ```
> @indiegamedevs gamedev
> https://t.me/c/1234567890/4350 3d
> @ai_news ai-startups
> ```

Получив список, собери строку `WATCHED_GROUPS_INIT` в формате, который
ожидает Telegram Watcher: `<ссылка>|<topic_hint>;...`

Для примера выше получится:

```
WATCHED_GROUPS_INIT=@indiegamedevs|gamedev;https://t.me/c/1234567890/4350|3d;@ai_news|ai-startups
```

Обнови `.env` — допиши эту строку.
**✓ Шаг 8 выполнен**, когда в `.env` непустой `WATCHED_GROUPS_INIT`.

---

## Шаг 9 — Telegram login (одноразовый)

Это авторизует user-аккаунт в Telegram и создаст session-файл. Запусти:

```bash
python -m app.setup_session
```

Telegram пришлёт **код логина** в само приложение Telegram. Скрипт
`setup_session` напечатает одно из двух:

- **TTY-режим (обычно):** спросит `Enter Telegram login code:`. Спроси
  код у пользователя в чате, потом введи его в работающий процесс.
- **Non-TTY режим (иногда в контейнерах / не-интерактивных shell'ах):**
  напечатает `NEED_CODE: /path/to/tmpfile` и будет ждать. Тогда спроси
  код у пользователя и запиши в файл:
  `echo "12345" > /path/to/tmpfile`.

**Если у аккаунта 2FA:** аналогичный поток для 2FA cloud password.
Используй callback через temp-файл (`NEED_PASSWORD: /path/...`) — **не**
прокидывай пароль через обычный stdin pipe; он попадёт в shell history.

Если получишь `Code expired` — просто перезапусти команду; Telegram
пришлёт свежий код.

При успехе будут строки типа `Session written to
data/sessions/telegram_watcher.session` и `Resolved N groups`.

**✓ Шаг 9 выполнен**, когда session записан и seed-группы зарезолвлены.

## Шаг 11 — Запустить supervisor

Это и есть 24/7-процесс: realtime listener + scheduler + dashboard в
одном event loop.

```bash
python -m app.supervisor
```

Жди такие лог-строки (появятся в течение ~5 секунд):

```
Supervisor: logged in id=<твой_id> username=@<ты> realtime_ai=True
Realtime group handler armed
Scheduler armed: daily_digest at 08:00 Europe/Kyiv
Dashboard at http://127.0.0.1:8000
```

Потом открой <http://127.0.0.1:8000> в браузере пользователя. **Feed**
должен загрузиться. Сначала может быть пусто — это нормально, сообщения
появляются по мере публикации в watched-группах.

Чтобы остановить supervisor: `Ctrl-C`.

Чтобы запустить постоянно:

```bash
nohup .venv/bin/python -m app.supervisor > watcher.log 2>&1 &
```

(Linux/macOS; для Docker или systemd — см. `INSTALL.md`.)

**✓ Шаг 11 выполнен**, когда dashboard грузится в браузере.

## Шаг 12 — Что дальше

Скажи пользователю кратко:

- **Логи:** `tail -f watcher.log` (если запускал через `nohup`), иначе
  смотри терминал, где работает supervisor.
- **Добавить/удалить группы позже:** отредактируй `WATCHED_GROUPS_INIT`
  в `.env` и перезапусти `python -m app.setup_session`. Он перечитает
  env-список и upsert'нет группы в БД.
- **Подкрутить фильтр релевантности:** редактируй
  `system_prompts/group_relevance_filter.md`. Это единственный AI-промпт.
  После правки перезапусти supervisor, чтобы изменения вступили в силу
  для realtime-классификации.
- **Daily digest:** запускается автоматически раз в день в час, который
  задал `DIGEST_CRON_HOUR` в `.env` (по умолчанию 08:00 в `LOCAL_TZ`).
  Можно вручную: `python -m app.run_daily_digest`.
- **Production / VPS deployment:** см. `INSTALL.md` § 3b (Docker) и
  раздел Security про dashboard auth-token и публикацию наружу.
- **Privacy reminder:** `.env`, `data/sessions/*.session` и
  `data/telegram_watcher.db` уже в `.gitignore`. Не делиться ими. Если
  подозреваешь, что аккаунт скомпрометирован — `INSTALL.md` §
  Security → "If you suspect your account is compromised".

В конце финальная самопроверка: попроси подтвердить, что dashboard
открывается и сообщения в feed'е появляются (может занять несколько
минут, если группы тихие). 🎉

## Failure recovery cheatsheet

Если какой-то шаг упал и пользователь хочет начать с нуля:

```bash
rm -f .env
rm -rf data/
deactivate 2>/dev/null
rm -rf .venv
```

Потом перезапусти этот workflow с шага 1.
